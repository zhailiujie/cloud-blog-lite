# 响应式适配开发记录

本文档记录 Web 端电脑/手机响应式适配的分阶段改造进度。每完成一步后更新对应状态，便于后续继续推进。

## 目标

- 导航页在手机和电脑端都具备良好浏览体验。
- 登录页在小屏幕下不溢出、不压缩变形。
- 后台管理页在手机端至少可用，后续再优化为更好的移动端交互。
- 尽量只调整布局和样式，不改变业务逻辑。

## 阶段 1：全局移动端基础样式与登录页适配

完成时间：2026-05-26

### 已完成内容

- 为 `max-width: 640px` 增加基础移动端规则。
- 限制 `body` 横向溢出，避免手机端出现横向滚动条。
- 缩小 `.page` 在手机端的左右边距。
- 登录卡片 `.auth-card` 改为按视口宽度自适应：`min(380px, calc(100vw - 32px))`。
- 登录页背景装饰在手机端缩放为 `110vw`，避免背景图形过小或溢出异常。
- `PageHeader` 在手机端改为纵向排列，减少标题和按钮挤压。
- 手机端标题字号略微缩小。
- 搜索输入框、下拉筛选框在手机端宽度改为 `100%`。
- 表单弹窗 `.form-modal` / `.form-modal.large` 在手机端宽度限制为 `calc(100vw - 24px)`。

### 涉及文件

```text
apps/web/src/styles/main.css
```
